# Start ADD iteration

Use the **ADD 3.0 step-by-step** skill to:

1. Detect the current ADD state from existing docs (drivers, elements, last documented step).
2. Propose the **next** ADD step (Part 1 or Part 2) and list the artifacts to create or update and their paths.
3. Wait for user confirmation, then create/update only the agreed artifacts.
4. After that, suggest the following step or ask whether to continue.

Keep the flow human-guided: one step at a time, confirm before writing.
