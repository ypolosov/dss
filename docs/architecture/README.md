# Architecture Documentation

## GT Platform - Architecture

This directory contains architecture documentation following the ADD 3.0 methodology.

## Structure

| Directory | Description |
|-----------|-------------|
| [c4/](c4/) | LikeC4 diagrams (C4 model as code) |
| [adrs/](adrs/) | Architecture Decision Records (MADR v3) |
| [drivers/](drivers/) | Architecture drivers (use cases, QAs, constraints, concerns) |
| [decisions/](decisions/) | Design decisions log |
| [views/](views/) | Architecture views |
| [iterations/](iterations/) | ADD iteration logs |

## Key Documents

- [Utility Tree](utility-tree.md) - Quality attribute priorities
- [Kanban Board](kanban.md) - Design progress tracking

## Getting Started

1. Run `/sa-init` to initialize this structure
2. Run `/ba-qaw` to define quality attributes
3. Run `/ba-utility-tree` to prioritize quality attributes
4. Run `/sa-iterate` to start the first ADD iteration
